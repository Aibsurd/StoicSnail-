# SESSION.md — Управление сессиями и контекстом

> Как я управляю контекстом чтобы не переполняться.

---

## Проблема

LLM имеют ограниченное "контекстное окно" — максимум токенов которые видят за раз.

Когда сессия длинная:

- Много сообщений → занято больше места
- Много tool results → ещё больше
- Достигаем лимита → ошибка или обрезание

---

## Как OpenClaw решает

### 1. Auto-compaction (автоматическая)

Когда близко к лимиту:

- Старые сообщения → суммаризируются в краткое "резюме"
- Недавние сообщения → остаются как есть
- История сжимается → место освобождается

**Настроено:**

- `mode: safeguard` — строже сохраняет recent context
- `reserveTokens: 30000` — место для генерации ответа
- `keepRecentTokens: 80000` — минимум recent context
- `maxHistoryShare: 0.5` — max 50% на историю

### 2. Memory flush (перед compaction)

Перед сжатием — я **записываю важное в файлы**:

- Решения → MEMORY.md
- События → memory/YYYY-MM-DD.md

Это гарантирует что **критичное не потеряется**.

### 3. Session pruning

Trim старых tool results из памяти (не из истории).

### 4. Daily reset

Сессия сбрасывается в 4 AM (gateway local time).

## Что я делаю для управления

### Признаки что контекст большой:

- `/status` показывает высокий % context
- Ответы начинают "забывать" ранний контекст

### Ручные команды:

| Команда               | Что делает                        |
| --------------------- | --------------------------------- |
| `/compact`            | Принудительная compaction сейчас  |
| `/compact Focus on X` | Compaction с фокусом на X         |
| `/status`             | Показать использование контекста  |
| `/new`                | Начать новую сессию (чистый лист) |
| `/reset`              | То же что /new                    |

### Когда использовать:

**`/compact`** — когда:

- Сессия была длинной но важно сохранить контекст
- Хочешь "очистить" не теряя историю

**`/new`** — когда:

- Хочешь чистый лист
- Предыдущая сессия была "мусорной"
- Начинаешь новую тему

---

## Моя стратегия

### Long sessions (>1 hour)

1. Периодически делаю `/compact` если вижу что контекст растёт
2. Записываю важное в файлы **до** compact
3. После compact — контекст очищен, но файлы содержат всё важное

### Project-based sessions

1. `/new` когда начинаю новый проект
2. Старый проект → записан в PROJECTS.md
3. Новый контекст для нового проекта

### Daily workflow

1. Утром читаю memory/ вчерашний
2. Работаю
3. Вечером записываю итоги в memory/сегодня

---

## Метрики

Смотреть в `/status`:

```
Context: 82k/205k (40%)
```

Хорошо: < 60%
Внимание: 60-80%
Плохо: > 80%

---

## Предотвращение проблем

**Не накапливать мусор:**

- Не читать все файлы подряд
- Делать `/compact` если сессия была длинной
- Начинать `/new` для новых тем

**Файлы важнее памяти:**

- Важное → в файлы
- Контекст может сжаться, файлы остаются
