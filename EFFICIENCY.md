# EFFICIENCY.md — Оптимизация ресурсов

> Как я оптимизирую использование токенов, времени и памяти.
> Это важно потому что каждый запрос стоит денег.

---

## Основные принципы

### Токены

**Проблема:** Контекст ограничен. Каждый файл в промпте — это расход.

**Оптимизации:**
- Держу bootstrap файлы компактными
- Не читаю всё подряд — читаю то что нужно
- После использования — обновляю файлы а не добавляю новое

**Практика:**
- `MEMORY.md` — только существенное, < 5KB
- `AGENTS.md` — только operational, < 3KB
- Детали — в специализированных файлах

### Время

**Проблема:** Slow is smooth, smooth is fast. Но есть предел.

**Оптимизации:**
- Медленный анализ → быстрое выполнение (не наоборот)
- Читать один раз — правильно
- Делать один раз — правильно

### Память (файловая)

**Проблема:** Файлы накапливаются. Если не чистить — бардак.

**Оптимизации:**
- `archive/` для старого
- Еженедельный review
- Удаление мусора

---

## Практические правила

### При чтении
1. Проверить — нужен ли этот файл сейчас?
2. Прочитать минимально необходимое
3. Не читать файл целиком если нужна одна секция

### При записи
1. Записывать минимально необходимое
2. Не дублировать информацию
3. Обновлять а не добавлять

### При исследовании
1. Сначала оценить — нужен ли deep research?
2. Простой вопрос → web_search
3. Сложный → deep_research.sh с простым query

## Реальная стоимость (MiniMax-M2.7 via OpenRouter)

| Параметр | Цена |
|----------|------|
| Input tokens | $0.30 / 1M |
| Output tokens | $1.20 / 1M |
| Context window | 204,800 tokens |

**Примеры:**
- Маленькая сессия (10K in, 2K out) ≈ $0.003 + $0.0024 = ~$0.005
- Большая сессия (100K in, 50K out) ≈ $0.03 + $0.06 = ~$0.09
- Дневная работа (~500K total tokens) ≈ $0.15–0.50

**Output в 4 раза дороже input.** Это значит: компактные ответы экономят деньги.
