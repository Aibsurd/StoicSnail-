# STANDING-ORDERS.md — Постоянные полномочия агента

> Этот файл определяет что я могу делать автономно без явного запроса Operator.
> Загружается каждую сессию через AGENTS.md.

---

## Program 1: Workspace Integrity

**Полномочия:** Поддерживать workspace в рабочем состоянии  
**Триггер:** checkpoint-monitor cron (каждые 30 мин) + daily-self-test  
**Approval gate:** None для рутинных операций  
**Эскалация:** Если FAIL в self-test → уведомить Operator  

### Что могу делать автономно:
- `git add + commit + push` при наличии изменений
- Обновлять `memory/YYYY-MM-DD.md` событиями дня
- Исправлять очевидные баги в скриптах (не в логике)
- Создавать daily log если не существует
- Запускать self-test и интерпретировать результаты

### Что НЕ делаю без запроса:
- Изменять MEMORY.md (только при compaction или явном запросе)
- Удалять файлы (только `trash` или явный запрос)
- Изменять openclaw.json config
- Устанавливать новые npm пакеты
- Создавать новые cron jobs

## Program 2: Research on Demand

**Полномочия:** Выполнять исследования без дополнительных вопросов  
**Триггер:** Запрос Operator на исследование любой темы  
**Approval gate:** None для web search + fetch  
**Эскалация:** Если тема требует приватных данных → спросить  

### Что могу делать автономно:
- Web search через SearXNG (multiple queries, not single)
- Fetch URLs для углублённого изучения
- Запускать `deep_research.sh` для структурированных отчётов
- Сохранять результаты в `archive/research/`
- Создавать sub-agents для параллельного исследования

### Стандарт исследования:
1. Минимум 3 поисковых запроса (не один compound)
2. Минимум 2-3 источника с fetch для деталей
3. Критическая оценка результатов
4. Сохранение в файл если > 500 слов

## Program 3: Autonomous Self-Improvement

**Полномочия:** Улучшать собственные инструменты и логику  
**Триггер:** Обнаружение проблемы или явный запрос аудита  
**Approval gate:** Изменения в скриптах — ok. Config — спросить.  
**Эскалация:** Если изменение затрагивает внешние системы → спросить  

### Что могу делать автономно:
- Исправлять баги в `scripts/*.sh` и `tools/runtime/*.mjs`
- Добавлять новые функции в existing tools
- Создавать новые инструменты в `tools/runtime/`
- Обновлять документацию (CAPABILITIES.md, TOOLS.md, MEMORY.md)
- Добавлять записи в agenda/think через db

### Что НЕ делаю без запроса:
- Не изменяю SOUL.md, IDENTITY.md (это конституция)
- Не изменяю AGENTS.md без audit запроса
- Не устанавливаю npm packages без явного решения

## Program 4: Session Continuity

**Полномочия:** Обеспечить преемственность между сессиями  
**Триггер:** Начало каждой сессии  
**Approval gate:** None  

### Стандартный startup:
1. Читать MEMORY.md (уже в context)
2. Читать memory/YYYY-MM-DD.md (сегодня + вчера)
3. Читать PROJECTS.md + INTENTS.md
4. Если есть открытые задачи в agenda → проверить
5. Если есть blocker в think → оценить

### При /new или /reset:
- session-memory hook автоматически сохранит контекст
- При следующем старте прочитать файл из memory/
